🛠️ After Login — One-Time API Setup & Troubleshooting

Enable API Connections in Gateway UI (do this once inside VNC)

  1. 1 In the VNC window, click Configure → API → Settings
  2. 2 Check ✅ "Enable ActiveX and Socket Clients"
  3. 3 In Trusted IPs, add 172.0.0.0/8 (Docker network), or uncheck "Allow connections from localhost only"
  4. 4 Click OK, then come back here and click ⚡ Connect

💡 IB Gateway saves this setting — you only need to do it once per installation.

Common Problems

🔁 Login dialog keeps reappearing

2FA timed out — complete it within ~60 seconds of it appearing in the VNC window. If stuck in a loop, restart the container:

docker-compose restart ib-gateway

⚠️ API connect fails after login

Most common cause: "Enable ActiveX and Socket Clients" is not checked — follow the 4 steps above. Also look for an "Accept API connection?" popup inside the VNC window and click Allow.

🔵 Existing session / another device logged in

Log out of all other IBKR sessions first, or dismiss the "Existing session detected" dialog inside the VNC window by selecting Exit Application on the other session.
